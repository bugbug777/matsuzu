## 網頁的骨架：初識 HTML

### 什麼是 HTML？

HTML，全名為 **H**yper**T**ext **M**arkup **L**anguage（超文本標記語言）。聽到「標記語言」，你可能會覺得有點陌生，但它的概念其實非常直觀。

<!--more-->

簡單來說，HTML 不是一種傳統的「編程語言」（Programming Language），它不能用來執行複雜的邏輯運算；它是一種**描述性語言**，專門用於**結構化**網頁內容。

想像一下人體：

  * **HTML** 就像是**骨架**（Skeleton），它決定了內容的結構和層次，例如哪裡是頭部、哪裡是身體、哪裡是段落、哪裡是圖片。
  * **CSS** (Cascading Style Sheets) 就像是**皮膚和服裝**，負責美化骨架，決定顏色、字體、佈局和外觀。
  * **JavaScript** 就像是**靈魂**，負責讓網頁動起來，處理互動、計算和數據。

作為一個新手，你需要先掌握好這個「骨架」，才能放心地進行下一步的裝飾和互動。

### HTML 的歷史與重要性

HTML 自 1990 年代初誕生以來，歷經多次迭代，目前廣泛使用的是 **HTML5**。HTML5 不僅增強了語義性（Semantic），還引入了許多多媒體和應用程式介面的支援，使其成為現代網頁開發的標準。

**為什麼它如此重要？**

  * **內容呈現的核心：** 無論你看到的多麼華麗的網站，底層一定是 HTML 在支撐。瀏覽器（如 Chrome, Firefox, Safari）唯一能直接理解並渲染的語言就是 HTML。
  * **搜尋引擎優化 (SEO)：** 良好的 HTML 結構（尤其是使用正確的語義標籤）能幫助 Google 或其他搜尋引擎更好地理解你的網頁內容，從而提升你的網站排名。
  * **輔助技術支援：** 對於使用螢幕閱讀器等輔助技術的使用者來說，清晰、結構化的 HTML 是他們理解網頁內容的唯一途徑。

## HTML 的基本元素和結構

HTML 文件是由一連串的「**元素**」（Elements）所構成。

### 核心概念：標籤 (Tags) 和元素 (Elements)

一個 HTML **元素**通常由一對**標籤**（Tags）組成：一個**開始標籤**（Opening Tag）和一個**結束標籤**（Closing Tag）。

> $>$ **範例：**
> `<p>` 是開始標籤，`</p>` 是結束標籤。中間的文字就是內容。
>
> **一個完整的元素：** `<p>這是一個段落的內容。</p>`

有些元素是「**空元素**」（Void Elements），它們不需要結束標籤，例如換行標籤 `<br>` 和圖片標籤 `<img>`。

### 屬性 (Attributes)

標籤可以擁有**屬性**（Attributes），這些屬性提供了元素的額外資訊，或者用來配置元素的行為。屬性通常寫在開始標籤內，格式為 `屬性名="值"`。

> $>$ **範例：**
> 我們用 `<img>` 標籤來顯示圖片，`src` 屬性用來指定圖片的路徑，`alt` 屬性用來描述圖片內容（對 SEO 和無障礙很重要）。
>
> `<img src="logo.png" alt="公司標誌">`

### 頁面的基本結構

所有的 HTML 文件都必須遵循一個基本結構：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
    <meta charset="UTF-8">
    <title>我的第一個網頁</title>
    </head>
<body>
    <h1>歡迎來到我的網站</h1>
    <p>這是網頁的主體內容。</p>
    </body>
</html>
```

| 標籤 | 說明 |
| :--- | :--- |
| `<!DOCTYPE html>` | **文件類型宣告**。它告訴瀏覽器這是一個 HTML5 文件。 |
| `<html>` | **根元素**。所有其他元素都必須包含在這個標籤內。`lang` 屬性指定語言。 |
| `<head>` | **頭部**。包含網頁的**元數據**（Metadata），例如字符集、網頁標題、引用的 CSS 和腳本等。這些內容通常不會直接顯示給使用者。 |
| `<body>` | **主體**。包含所有在網頁上**可見的內容**，例如文字、圖片、連結、表格等。 |
| `<meta charset="UTF-8">` | 設定字符編碼，`UTF-8` 是國際通用的標準，確保你的文字能正確顯示。 |
| `<title>` | 設定顯示在瀏覽器標籤頁上的**網頁標題**。 |

## 常見的 HTML 語義化標籤

HTML5 強調「**語義化**」（Semantic），意思是使用標籤時，不僅要考慮它顯示出來的效果，更要考慮這個標籤**代表的意義**。

| 標籤類別 | 標籤 | 語義上的意義 (功能) | 顯示效果 (默認) |
| :--- | :--- | :--- | :--- |
| **標題** | `<h1>` 到 `<h6>` | 代表內容的**標題層次**，`<h1>` 是最高級別（一頁通常只有一個）。 | 字體由大到小，粗體。 |
| **段落** | `<p>` | 代表一段**獨立的文字段落**。 | 段落間有間距。 |
| **列表** | `<ul>` / `<ol>` / `<li>` | **無序列表** (`<ul>`)、**有序列表** (`<ol>`) 和**列表項** (`<li>`)。 | 圓點/數字符號。 |
| **連結** | `<a>` | **錨點**（Anchor），用來**連結**到其他頁面或資源。 | 藍色帶底線。 |
| **媒體** | `<img>` | **圖像**。 | 顯示圖片。 |
| | `<video>` / `<audio>` | 嵌入**影片**或**音訊**。 | 顯示播放器控制項。 |
| **結構** | `<header>` | 網頁的**頁首**（通常包含 Logo、導航）。 | 無特殊樣式，常搭配 CSS。 |
| | `<nav>` | 網站的**導航連結區**。 | 無特殊樣式。 |
| | `<main>` | 頁面的**主要內容**區塊（一頁通常只有一個）。 | 無特殊樣式。 |
| | `<footer>` | 網頁的**頁尾**（通常包含版權、聯絡資訊）。 | 無特殊樣式。 |
| | `<section>` / `<article>` | 用來劃分**獨立的主題或內容區塊**。 | 無特殊樣式。 |

### 避免使用 Div 和 Span 的誤區

在過去，開發者可能會大量使用不帶語義的 `<div>`（區塊容器）和 `<span>`（行內容器）來構建整個網站。

**作為新手導師，我強烈建議你：**

> **盡可能使用語義化的標籤！**
>
> 只有當找不到合適的語義標籤時，才考慮使用 `<div>` 來純粹地劃分結構，或使用 `<span>` 來標記行內文字（通常是為了套用 CSS 樣式）。

使用語義標籤，不僅讓你的程式碼更清晰易懂，對於未來的維護和 SEO 都有極大的益處。

## 結語與下一步

恭喜你！你已經掌握了 HTML 的核心概念：它是一種用標籤來結構化內容的語言。

記住，HTML 是**內容**，它決定了「**有什麼**」。至於「**長什麼樣子**」，那是 CSS 的職責；而「**會做什麼**」，則是 JavaScript 的任務。

**現在，你已經準備好寫下你的第一行 HTML 程式碼了！**
